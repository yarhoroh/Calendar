# Доработка e-mail — план

## ✅ Сделано (последнее)
- **Перевод письма прямо в просмотре.** Кнопка «Перекласти» + выпадающий список целевого языка рядом с Details. Берём текстовые узлы письма (`TreeWalker`, нумеруем), шлём в встроенную ИИшку (изолированный one-shot `agyAskRaw` — свежий conversation, без calendar-guard), получаем JSON `{номер: перевод}`, подменяем `nodeValue` узлов на месте. ИИ сам определяет исходный язык. «Оригінал» — мгновенный возврат (оригиналы сохранены). Структуру/стили/картинки письма не трогаем.

## 🔜 Следующий шаг — контекст почты для чата ИИ
Чтобы при общении в чате (когда я в почте) ИИ понимал ситуацию и мог помогать по письму:
- В `aiContext()` (main) добавлять:
  - активный вид = `mail` (а не календарь);
  - подключённые почтовые аккаунты (адреса);
  - текущая папка и выбор (список / открытое письмо);
  - **текстовое тело** открытого письма (`text`, без HTML/картинок) + тема/от/кому.
- Поток: рендер сообщает текущее состояние почты (через payload `ai:send` или отдельный сеттер контекста), main подмешивает это в промпт чата.
- Важно: слать **только текст**, не HTML и не картинки.

## 🧩 Большой план — AeTask для e-mail (правила/задачи по почте)
По аналогии с блоком **AI-tasks** в настройках — добавить отдельный блок **AeTask.email**:
- В настройках — список email-правил: создавать/редактировать/удалять вручную; **ИИ тоже может их добавлять**.
- У правила: условие (параметры письма — от кого / тема содержит / категория / есть вложение / ключевые слова и т.п.), цель (язык/аккаунт), действие.
- Триггер: при **поступлении новых писем** (поллинг уже есть) ИИ читает их **текстовое тело**;
  - если письмо подходит под параметры правила → выполнить действие:
    - выдать уведомление (голос / сообщение у трея — как у обычных AeTask);
    - **создать задачу/заметку в календаре**;
    - (в перспективе) ответить / пометить / переместить.
- Использовать те же команды/функции, что и ИИ для календаря (единый командный интерфейс), чтобы ИИ мог «дальше выполнять работу внутри календаря».

### Открытые вопросы к реализации
- Где хранить правила (рядом с `ai_tasks` в БД / отдельная таблица `mail_rules`).
- Как ИИ матчит письмо к правилу: детерминированный фильтр (от/тема/вложение) + опционально ИИ-проверка по «смыслу».
- Анти-спам: не дёргать ИИ на каждое письмо без нужды (сначала дешёвый фильтр, потом ИИ).
